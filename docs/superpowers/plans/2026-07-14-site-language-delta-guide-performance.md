# 网站中文默认、Delta 引导与性能优化实施计划

日期：2026-07-14  
状态：已完成  
对应规格：[网站中文默认、Delta 引导与性能优化规格](../specs/2026-07-14-site-language-delta-guide-performance-design.md)

## 1. 目标

在不改变现有视觉风格、Delta 查询行为和本地数据边界的前提下，完成默认中文、Delta 返回与引导，以及个人网站和美颜相机的性能优化。

## 2. 执行规则

- 每项行为先写测试并确认因缺少新行为而失败，再做最小实现使其通过。
- `pp-tools` 是工具源码；`My/tools/pp-tools` 只由构建同步生成，不手工修改。
- 不修改 Companion、OCR、玩家数据、通信协议或整体 UI。
- 不提交、不推送；最终由用户在 GitHub Desktop 中处理。

## 3. 预计文件范围

### `My`

- 新建：`js/site-runtime.js`、`tests/site-runtime.test.js`。
- 修改：`index.html`、`js/app.js`，必要时小范围修改 `css/style.css`。

### `pp-tools`

- 新建：`DeltaUsageGuide.jsx` 及测试、相机调度辅助模块及测试。
- 修改：Delta 命令栏、主页面、三语文案和样式；相机流、识别运行时、相机页面及对应测试。
- 生成：`frontend/dist-embed`，随后同步到 `My/tools/pp-tools`。

## 4. 实施任务

### 任务 1：记录基线

1. 确认两个仓库工作区状态，只保留已通过的规格和本计划文档。
2. 运行现有测试与构建，区分原有失败和本轮回归。
3. 在真实浏览器记录当前页面帧间隔、长任务和相机显示帧率；无法获得摄像头权限时明确记录限制。

### 任务 2：默认中文

1. **RED**：测试无记录、无效记录回退中文，有效中/英/日选择保持不变。
2. **GREEN**：增加可测试的语言解析函数，并让 `app.js` 与 `html lang` 使用该结果。
3. **VERIFY**：运行语言测试和全部 `My` 测试，确认切换语言仍可持久保存。

### 任务 3：Delta 返回入口

1. **RED**：测试顶部显示“返回工具中心”按钮，点击后调用站内导航且支持键盘操作。
2. **GREEN**：把明确返回按钮接入 Delta 顶部；嵌入模式返回工具中心，独立页面返回 PP Tools 首页。
3. **VERIFY**：运行命令栏、主页面和嵌入路由测试，检查桌面与窄屏不重叠。

### 任务 4：Delta 简洁引导

1. **RED**：测试未连接、待配对、已连接和任务执行时显示正确的下一步；测试三语键完整。
2. **GREEN**：新增状态化折叠引导，覆盖启动、配对、游戏准备、查询期间勿操作和手动上传备用方式。
3. **VERIFY**：运行引导、连接和主页面测试，确认引导不会遮挡查询栏与结果区。

### 任务 5：个人网站动画性能

1. **RED**：测试隐藏页面、沉浸式工具详情和减少动态效果模式会暂停装饰动画，恢复可见后允许继续。
2. **GREEN**：统一管理背景循环生命周期；停止无效循环；降低粒子画布像素与数量开销；避免重复写入相同样式。
3. **VERIFY**：运行 `My` 测试，并在首页、滚动页面和工具详情中测量帧间隔与长任务，截图核对视觉风格。

### 任务 6：美颜相机性能

1. **RED**：测试高帧率约束、能力不足回退、画面帧调度和识别自适应降级。
2. **GREEN**：优先请求摄像头最高可用帧率；使用真实视频帧驱动渲染并保留兼容回退；按识别耗时动态调整检测间隔。
3. **VERIFY**：运行相机测试和完整前端测试；检查相机启动、切换、关闭、美颜、手势和拍照不回归。

### 任务 7：构建、同步与完整验证

1. 运行 `pp-tools/frontend` 全部测试和生产构建。
2. 运行现有同步脚本，把嵌入构建更新到 `My`，并检查生成文件一致性。
3. 运行 `My` 全部测试和 `node scripts/check-site.mjs`。
4. 在真实浏览器验证中文默认、语言持久化、Delta 返回与引导、桌面/移动布局、页面动画和相机降级。
5. 检查两个仓库差异、文档状态和敏感内容，不提交或推送。
