# 本地生产运行实施计划

日期：2026-07-11  
状态：执行中

## 目标

将前端页面、后端接口和本地数据收敛为一个可持续运行入口，为后续接入独立域名或公网部署保留同源结构。

## 实施步骤

1. 先增加后端测试，覆盖前端静态文件、工具子路由回退和健康检查。
2. 调整 Flask 应用工厂，由后端统一提供 `frontend/dist` 和现有 `/api` 接口。
3. 增加 Windows 启动脚本，检查环境变量、数据库目录、前端构建产物和 Python 环境。
4. 使用生产服务进程启动单一端口，并在启动前输出健康检查结果。
5. 增加数据库备份脚本，生成带时间戳的副本，不覆盖历史备份。
6. 更新中文运行说明、配置示例和项目自检脚本。
7. 执行前后端全部测试、生产构建、单端口浏览器验收和仓库检查。

## 验收标准

- 一个本地地址同时打开首页、三个工具、管理端和接口。
- 工具子路由刷新后仍能返回网页，不出现 404。
- 缺少配置或构建产物时，启动脚本明确指出问题。
- 数据库目录自动创建，备份脚本不会覆盖现有文件。
- 密码、会话密钥和本机数据库不进入仓库。
- 自动测试、生产构建、健康检查和真实浏览器验收全部通过。
