# 个人网站工具中心嵌入设计

日期：2026-07-11  
状态：已完成

## 1. 背景

个人网站当前的 `Software` 页面使用静态项目卡片展示三个软件，只能提供预览、下载或源码链接。PP Tools 已经拥有更完整的工具中心界面，并提供 Delta 战绩分析、手势美颜相机和三平方奶茶店小程序源码下载。

本次改造需要让个人网站的 `Software` 页面采用 PP Tools 的完整视觉和功能，同时保留个人网站现有的主页、日记、关于、顶部导航和语言切换。PP Tools 仍是工具功能的唯一源码，不能在两个仓库分别维护两份工具代码。

## 2. 目标

1. 个人网站的 `Software` 正文完全替换为 PP Tools 工具中心。
2. 个人网站顶部导航始终保留，不显示 PP Tools 独立站点的顶栏和页脚。
3. Delta 战绩分析和手势美颜相机在个人网站内部打开，不跳转到独立站点。
4. 工具中心、两个网页工具和奶茶源码说明支持中文、英文、日文。
5. 使用 hash 路由适配当前 GitHub Pages，刷新和前进后退不产生 404。
6. PP Tools 独立网站继续正常运行，个人网站通过构建产物复用同一份工具源码。
7. 本阶段保证 Delta 在本机后端启动时可用；公网识别后端部署留到下一阶段。

## 3. 非目标

1. 本阶段不部署 Delta 公网后端。
2. 不把奶茶小程序改写成浏览器版本。
3. 不重写个人网站的主页、日记、关于和顶部导航。
4. 不把个人网站整体迁移到 React。
5. 不使用 iframe 嵌入 PP Tools。
6. 不删除 PP Tools 独立站点或改变其现有公开路由。

## 4. 总体架构

### 4.1 代码所有权

PP Tools 继续使用 React，并作为工具中心和工具功能的唯一源码。个人网站继续使用现有的 HTML、CSS 和 JavaScript，只负责外层导航、语言状态、hash 路由和嵌入容器。

PP Tools 新增嵌入构建入口，导出统一挂载接口。构建完成后，由同步脚本把固定白名单内的 JS、CSS、图片、相机运行资源和奶茶 ZIP 复制到个人网站专用目录。

### 4.2 嵌入接口

嵌入包提供 `mountToolCenter(container, options)`，其中 `options` 至少包含：

- `language`：当前语言，允许 `zh`、`en`、`ja`。
- `route`：当前工具路由。
- `apiBaseUrl`：Delta 后端地址。
- `assetBaseUrl`：嵌入资源根地址。
- `onNavigate(route)`：工具内部请求切换路由时调用。

挂载函数返回控制器，至少支持：

- `update(nextOptions)`：语言、路由或配置变化时更新界面。
- `unmount()`：离开 Software 页面时卸载 React 根节点并释放运行资源。

个人网站不直接操作 React 内部组件，也不复制工具业务逻辑。

### 4.3 路由

个人网站统一管理以下 hash：

- `#software`：工具中心。
- `#software/delta-force`：Delta 战绩分析。
- `#software/beauty-cam`：手势美颜相机。
- `#software/milk-tea`：奶茶小程序源码说明与下载。

工具内部导航通过 `onNavigate` 更新 hash。浏览器前进、后退或手动修改 hash 时，个人网站解析路由并调用嵌入控制器的 `update()`。未知的 `#software/*` 路由回退到 `#software`。

## 5. 页面与视觉设计

### 5.1 个人网站外层

个人网站的顶部导航、语言切换和背景装饰保持不变。进入 Software 后仍能直接切换主页、日记和关于页面。

`view-software` 改为工具中心专用容器，移除原来的项目卡片、胶带装饰和图片预览弹窗。该容器允许工具中心使用约 1120 像素的内容宽度，并为固定顶部导航预留安全间距。

### 5.2 工具中心

工具中心保留 PP Tools 当前的主要视觉：

- 柔和的粉色与青色背景层次。
- 居中的工具箱标题和入口按钮。
- 三个交错排列的大图工具区。
- 工具状态、功能标签和主要操作按钮。

嵌入模式不显示 PP Tools 自己的站点顶栏和“返回个人网站”页脚，避免出现两套导航。

### 5.3 工具详情

Delta 和手势相机详情页保留现有功能布局，但增加统一的“返回工具中心”操作。返回操作只更新 hash，不重新加载整页。

奶茶详情页展示项目图片、功能说明、导入方法、运行模式和源码下载按钮，不在浏览器内模拟微信小程序。

### 5.4 响应式要求

- 桌面端工具展示区保持图文双列和交错顺序。
- 小于 880 像素时改为单列，图片在上、文字在下。
- 小于 560 像素时压缩标题、内边距和按钮尺寸。
- 固定顶部导航不得遮挡工具标题、操作按钮或错误提示。
- 页面不得出现横向溢出，文本优先换行。

## 6. 三语系统

PP Tools 新增集中式语言字典，覆盖：

- 工具中心标题、说明、状态、标签和按钮。
- Delta 上传、识别、档案、筛选、删除、状态和错误文案。
- 手势相机权限、加载、控制、滤镜、拍照、预览和错误文案。
- 奶茶源码说明、导入步骤、运行模式和下载文案。
- 通用返回、加载、重试和不可用状态。

个人网站切换语言后调用嵌入控制器的 `update()`，当前工具不刷新页面即可更新文字。用户文件名、识别结果、玩家昵称、日期和数值保持原始内容，不做翻译。

缺失翻译时回退到中文，并在开发测试中报告缺失键；生产界面不能显示语言键名。

## 7. 数据与运行流程

### 7.1 Delta

Delta 的本地档案继续保存在浏览器本地存储中。截图识别请求发送到 `apiBaseUrl` 指定的 PP Tools 本机后端。

本地启动入口同时提供：

- 个人网站静态服务。
- PP Tools 后端服务。
- 前端所需的后端地址配置。

后端未启动时，历史档案和本地管理功能仍可打开；截图识别显示当前语言对应的“后端未连接”状态和重新检测操作。

公网阶段只替换 `apiBaseUrl`，不修改页面结构和工具业务组件。

### 7.2 手势相机

相机只在进入详情页并由用户启动后申请权限。离开相机页面、离开 Software 或卸载嵌入包时，必须停止所有媒体轨道并释放视觉运行资源。

相机模型和静态资源通过 `assetBaseUrl` 加载，不能依赖 PP Tools 独立站点的根路径。

### 7.3 奶茶源码

奶茶 ZIP 随嵌入产物同步到个人网站。下载链接使用个人网站内的相对地址，并保留固定文件名 `sanpingfang-miniprogram-source.zip`。

同步和检查脚本必须确认 PP Tools 与个人网站中的 ZIP 校验值一致。

## 8. 构建与同步

PP Tools 增加独立的嵌入构建命令，不改变现有独立站点构建命令。嵌入产物写入临时构建目录，通过同步脚本复制到个人网站的固定目录。

同步脚本执行以下步骤：

1. 构建嵌入入口。
2. 校验必需的 JS、CSS、图片、相机资源和奶茶 ZIP。
3. 清理个人网站中上一版嵌入产物。
4. 只复制白名单文件。
5. 输出文件数量、总体大小和奶茶 ZIP 校验值。

生成目录只保存构建产物，不手工维护业务代码。同步命令不执行 Git 提交或推送。

## 9. 状态与错误处理

### 9.1 嵌入加载

- 首次加载显示与个人网站风格一致的三语加载状态。
- JS 或 CSS 加载失败时显示三语错误、重试按钮和返回工具中心操作。
- 重试不得重复创建多个 React 根节点。

### 9.2 Delta

- 后端未启动：保留本地档案，禁用当前识别任务并提供重新检测。
- 请求超时：结束忙碌状态，保留用户已选择的文件。
- 识别结果为空：显示明确提示，不创建空档案。
- 部分字段成功：保存可用字段并显示警告状态。

### 9.3 手势相机

- 权限拒绝、无摄像头、模型加载失败和运行中断使用不同错误文案。
- 失败后允许重新启动，不要求刷新整个网站。
- 页面卸载后不允许继续占用摄像头或执行识别循环。

### 9.4 奶茶下载

- ZIP 不存在或校验失败时，同步构建直接失败。
- 浏览器端下载入口不可用时显示错误，不使用无效占位链接。

## 10. 测试策略

### 10.1 PP Tools 自动测试

- 三种语言均能获取全部必需文案。
- 缺失翻译正确回退到中文。
- 嵌入模式不渲染独立站点顶栏和页脚。
- 工具中心、Delta、相机和奶茶路由正确切换。
- `mount`、`update` 和 `unmount` 生命周期正确。
- 离开相机页面时媒体轨道被停止。
- Delta 使用传入的 `apiBaseUrl`，并正确处理后端离线。
- PP Tools 独立站点现有测试继续通过。

### 10.2 个人网站自动测试

- `#software` 及三个子路由可以解析和回退。
- 切换语言会更新嵌入控制器。
- 离开 Software 会调用 `unmount()`。
- 工具内部导航只更新 hash，不打开新标签页。
- 旧的静态软件卡片和预览弹窗不再渲染。
- 嵌入资源和奶茶 ZIP 均存在。

### 10.3 构建与浏览器验收

- PP Tools 独立构建通过。
- 嵌入构建与同步脚本通过。
- 个人网站结构检查通过。
- 两个网站中的奶茶 ZIP 校验值一致。
- 三个仓库执行 `git diff --check` 无格式错误。
- 在真实浏览器中验证 1440×900 和 390×844。
- 验证四个主导航、三个工具路由、三语切换、前进后退和刷新。
- 验证 Delta 后端正常与未启动两种状态。
- 验证相机授权、拒绝权限和离开页面释放摄像头。

## 11. 验收标准

1. 个人网站 Software 页面在视觉上与 PP Tools 工具中心一致，同时保留个人网站顶部导航。
2. Delta 和手势相机在个人网站内部打开，网址使用约定的 hash。
3. 工具中心和工具详情的所有固定可见文案支持中、英、日三语。
4. PP Tools 独立网站功能和路由不回退。
5. Delta 本机后端启动时可以完成截图识别，未启动时有明确错误状态。
6. 手势相机离开页面后不再占用摄像头。
7. 奶茶源码可以下载，两个网站中的 ZIP 完全一致。
8. 桌面端和手机端没有导航遮挡、横向溢出或文字裁切。
9. 自动测试、构建、项目检查和格式检查全部通过。
10. 实施过程不自动提交或推送仓库。

## 12. 后续阶段

本阶段完成后，单独设计 Delta 公网后端部署，包括托管平台、域名、跨域规则、上传限制、日志、费用控制和健康检查。该阶段只替换个人网站的后端地址配置，不重新设计工具界面。
