# Delta Force 原版界面迁移设计

日期：2026-07-10
状态：已确认，可进入实施计划阶段

## 概要

使用 React 重建原版 Delta Stats 界面，替换当前通用风格的 Delta Force 上传页面。完整保留原版全屏深色工作台、信息层级、上传流程、花名册、玩家档案、数据对比表、雷达图和近期战绩布局，同时继续使用当前 Flask 截图分析接口和真实识别核心。

原版桌面客户端自动控制无法在浏览器中运行。顶部查询控件保留原有位置和外观，但功能调整为按照昵称或 UID 查询当前访客保存在浏览器中的花名册。

## 目标

- 保留原版 Delta Stats 的视觉风格和主要交互结构。
- 继续通过当前网页接口提供截图识别能力。
- 恢复原版花名册和完整玩家档案体验。
- 将玩家档案保存在每位访客自己的浏览器中，避免不同访客互相看到数据。
- 保持桌面端和移动端均可使用的响应式布局。

## 不在本次范围内

- 不控制访客的游戏客户端、桌面窗口、键盘或鼠标。
- 不提供原版校准流程。
- 不增加用户账号或服务器端玩家历史记录。
- 除满足现有接口契约所必需的调整外，不修改识别模型和解析规则。
- 不长期保留上传的截图。

## 页面结构

Delta Force 路由改为沉浸式页面，不再显示 `pp-tools` 共用的白色顶部导航栏。

### 顶部指挥栏

保留原版顶部指挥栏：

- `DELTA STATS` 品牌和 `烽火地带 · 战绩分析` 副标题。
- 昵称或 UID 查询输入框。
- 查询按钮。
- 档案数量摘要。
- 在原“校准”位置提供返回工具中心的链接。

查询时先在本地档案中精确匹配昵称或 UID 并打开对应档案。如果没有精确匹配，则使用输入内容筛选花名册并显示明确的空状态。

### 截图上传区

保留原版可折叠上传面板，并支持：

- 选择文件。
- 拖放图片。
- 粘贴剪贴板图片。
- 多张截图预览。
- 已选文件数量。
- 清空操作。
- 识别操作和进度状态。

上传面板在视觉上仍是顶部指挥栏的次要入口，但在粘贴或选择文件后自动展开。

### 花名册与玩家档案

使用原版双栏工作区：

- 左侧：按照最近更新时间排序的本地花名册和筛选输入框。
- 右侧：当前选中的玩家档案。

玩家档案保留：

- 可编辑昵称、称号和 UID 展示。
- KD、撤离率、段位、赚损比、总场次和总资产摘要条。
- 数据总览与排位赛对比表。
- 可编辑 KD 字段。
- 可用模式的雷达图。
- 详细统计数据。
- 近期战绩摘要和明细行。
- 标签、备注、保存和删除操作。
- 最后更新时间。

缺失值统一显示为 `—`，不能因为部分字段缺失而隐藏其他已识别结果。

## 组件边界

`DeltaForcePage` 负责页面级状态和接口调用。实现拆分为职责明确的模块或组件：

- `deltaRecordStore`：加载、校验、新增或更新、编辑、搜索和删除浏览器本地档案。
- `DeltaCommandBar`：品牌、花名册查询、档案数量和工具中心链接。
- `DeltaUploadPanel`：文件收集、预览、拖放、粘贴和识别状态。
- `DeltaRoster`：筛选输入框、档案摘要、选中状态和空状态。
- `DeltaDossier`：完整玩家档案展示和编辑操作。
- `DeltaRadar`：原版五边形雷达图。

页面不得复制原版 `/api/players`、`/api/auto-lookup`、任务轮询或校准接口调用，只使用当前 `/api/delta-force/analyze` 接口和本地档案存储。

## 本地档案模型

所有档案保存在一个带版本号的 `localStorage` 键中。每条档案包含：

- `id`：稳定的本地标识符。
- `nickname`：识别或手动修改后的昵称。
- `uid`：识别到的 UID。
- `title`：识别到的玩家称号。
- `tags`：访客维护的标签列表。
- `note`：访客维护的备注。
- `data`：完整标准化识别结果，包括原始 `home`、`overview`、`ranked` 和 `recent` 部分。
- `created_at`：首次保存时间。
- `updated_at`：最近一次识别或手动保存时间。

新增或更新档案时按照以下顺序匹配：

1. 非空 UID 完全一致。
2. 否则使用规范化后的非空昵称完全一致。
3. 否则创建新的本地档案。

本地存储内容损坏时回退为空花名册，任何异常存储值都不能导致页面无法渲染。

## 数据流

1. 访客选择、拖入或粘贴截图。
2. 页面校验至少存在一张图片。
3. 页面向 `/api/delta-force/analyze` 发送 multipart 请求。
4. 后端临时保存文件、运行识别、返回标准化结果和警告，然后删除文件。
5. 页面将结果转换为本地档案，并新增或更新浏览器存储。
6. 新增或更新的档案成为当前选中项，并移动到花名册顶部。
7. 手动编辑只更新本地档案。

预览图片使用的对象地址必须在清空、替换文件或页面卸载时释放。

## 错误处理

- 未选择图片：在上传面板内显示提示。
- 本地文件类型不支持：忽略该文件并提示只接受图片。
- 网络或服务器失败：保留现有花名册并显示稳定的接口错误。
- 部分识别成功：保存并展示已有字段，缺失值显示为 `—`，同时显示第一条警告。
- 无法识别截图：显示后端警告，不使用空结果覆盖现有档案。
- 本地存储损坏：重置为空的有效档案集合。
- 删除档案：删除当前本地档案前必须二次确认。

## 响应式行为

桌面端保留原版顶部指挥栏以及花名册与档案双栏布局。较窄屏幕下：

- 顶部指挥栏控件允许换行且不能被裁切。
- 花名册移动到玩家档案上方。
- 数据对比表和近期战绩行不能造成页面横向溢出。
- 次要战绩字段可以沿用原版响应式规则收起。
- 核心 KD、段位、撤离率和对局结果始终可见。

## 测试

前端自动测试覆盖：

- 渲染原版顶部指挥栏、上传面板、花名册和空档案状态。
- 选择截图并显示识别结果。
- 创建本地档案并在重新加载后恢复。
- 按昵称和 UID 搜索。
- 编辑并保存玩家档案。
- 确认后删除档案。
- 从损坏的本地存储中恢复。
- 接口失败和部分识别状态。

现有后端识别、临时文件清理和错误契约测试继续保持通过。

真实浏览器验收覆盖：

- 桌面端与原版界面对照。
- 浏览器支持本地文件上传时，完成选择文件和结果展示。
- 花名册搜索、选择、编辑和删除。
- 移动端无横向溢出。
- 浏览器控制台无错误或警告。

## 验收标准

- Delta 路由在视觉上恢复原版全屏深色工作台，不再使用通用白色工具页。
- 原版顶部指挥栏、折叠上传区、花名册和完整档案结构全部存在。
- 真实截图识别结果能够填充到原版档案界面。
- 浏览器本地档案可以跨刷新保留，并且只属于当前浏览器。
- 顶部昵称或 UID 查询可以打开或筛选本地档案。
- 服务器上的上传截图仍然只做临时处理。
- 桌面端和移动端浏览器验收通过。
- 所有前端和后端测试通过。
