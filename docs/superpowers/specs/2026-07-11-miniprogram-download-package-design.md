# 三平方奶茶店小程序源码下载设计

日期：2026-07-11  
状态：已完成

## 1. 背景

三平方奶茶店原项目是微信原生小程序，使用 WXML、WXSS、JavaScript、微信组件和微信云开发。普通浏览器无法准确还原微信开发者工具中的导航栏、组件行为、布局单位和运行环境，因此不再将浏览器改写版作为最终交付。

本规格覆盖旧设计中“奶茶店直接在浏览器使用”的要求。工具网站保留项目展示，但最终入口改为下载经过脱敏处理的微信小程序源码包。

## 2. 目标

1. 网站访客点击按钮即可下载完整小程序源码 ZIP。
2. 下载包可以直接导入微信开发者工具并编译。
3. 默认本地演示模式不依赖原作者的 AppID、云环境或云数据库。
4. 商品、购物车、下单、订单和管理页面可以在本地演示模式中体验。
5. 使用者可以配置自己的微信项目和云环境，切换到云开发模式。
6. 下载包不包含原作者的私有配置、云环境标识、Git 历史或本机文件。

## 3. 非目标

1. 不在普通浏览器中继续复刻小程序界面。
2. 不向下载者提供原作者的微信项目权限或云环境权限。
3. 不保证下载者无需微信开发者工具即可运行小程序。
4. 不代替下载者完成小程序注册、备案、审核和发布。
5. 本阶段不配置独立域名或公网部署。

## 4. 网站入口

工具首页中的三平方奶茶店项目卡调整为：

- 状态：`源码可下载`；
- 主按钮：`下载小程序源码`；
- 说明：使用微信开发者工具导入，内置本地演示模式；
- 下载文件：`sanpingfang-miniprogram-source.zip`；
- 项目预览图继续使用真实小程序界面截图。

公共导航中的奶茶店入口改为下载入口或定位到项目卡，不再进入 `/tools/milk-tea` 浏览器页面。旧浏览器路由不作为公开入口，也不在网站上宣称可直接使用。

## 5. 源码包结构

源码包以现有 `miniprogram-1` 项目为来源，生成独立的发布副本。压缩包根目录直接包含微信开发者工具可识别的文件，避免解压后出现多余嵌套目录。

主要内容包括：

- `app.js`、`app.json`、`app.wxss`；
- `pages` 页面目录；
- `components` 公共组件目录；
- `styles` 设计系统目录；
- `utils` 工具和数据服务目录；
- `cloudfunctions` 云函数目录；
- `tabbar` 图标资源；
- `project.config.json` 公共项目配置；
- `README.md` 中文导入和配置说明。

发布副本不包含：

- `.git`；
- `project.private.config.json`；
- 原作者 AppID；
- 原作者云环境 ID；
- 本机缓存、日志和临时文件；
- 与运行无关的编辑器私有配置。

## 6. 运行模式

### 6.1 本地演示模式

下载包默认启用本地演示模式。数据保存在微信开发者工具当前模拟器的本地存储中，不调用原作者的云函数或云数据库。

本地演示模式至少支持：

- 演示账号登录；
- 商品分类、搜索和详情；
- 杯型、甜度、冰量和加料选择；
- 购物车增删和数量调整；
- 创建订单和查看订单状态；
- 收藏、常购和最近浏览；
- 管理端查看订单；
- 商品新增、编辑、启停和库存演示；
- 门店设置和经营数据演示。

演示数据允许重置。重置操作必须有确认提示，防止误清空当前模拟器数据。

### 6.2 云开发模式

源码保留现有云函数和云数据服务。使用者需要完成以下配置后才能切换：

1. 使用自己的 AppID 导入项目；
2. 在微信开发者工具中开通云开发；
3. 创建自己的云环境；
4. 在统一配置文件中填写云环境 ID；
5. 上传并部署 `cloudfunctions` 中的云函数；
6. 创建说明文档列出的数据库集合；
7. 将运行模式从 `demo` 改为 `cloud`。

运行模式和云环境 ID集中在一个配置文件中，不要求使用者到多个业务文件逐个修改。

## 7. 数据适配

页面不直接决定使用本地数据还是云数据。新增统一数据适配层，对页面提供稳定方法，例如：

- 用户登录和账号管理；
- 商品读取和维护；
- 购物车及用户数据；
- 订单创建、查询和状态更新；
- 门店配置；
- 用户反馈；
- 经营数据。

本地演示适配器使用本地存储；云开发适配器继续调用微信云函数。页面只依赖适配层，切换模式时不修改页面业务逻辑。

## 8. 脱敏与安全

1. `project.config.json` 中不保留原 AppID，使用适合导入的公共占位配置。
2. `app.js` 中不保留原云环境 ID。
3. 私有项目配置不进入发布副本和 ZIP。
4. 打包前扫描 AppID、云环境 ID、密钥格式、绝对路径和私有配置文件。
5. 演示账号只用于本地数据，不代表真实线上账号。
6. 云开发模式不包含预设管理员真实密码，首次初始化时由使用者自行设置。
7. 网站只提供生成后的公开 ZIP，不读取或打包原项目的 Git 目录。

## 9. 打包流程

项目提供可重复执行的打包脚本：

1. 从 `miniprogram-1` 读取受 Git 跟踪的源码文件；
2. 复制到临时发布目录；
3. 写入脱敏后的公共项目配置和运行配置；
4. 排除私有配置和无关文件；
5. 执行敏感内容扫描；
6. 生成 `sanpingfang-miniprogram-source.zip`；
7. 将 ZIP 输出到网站公开下载目录；
8. 输出文件数量、压缩包大小和校验值。

每次原小程序源码更新后，可以重新执行脚本生成新版本，不手工维护两份完整源码。

## 10. 错误处理

- 下载文件不存在时，网站按钮显示“下载暂不可用”，不跳转到空链接；
- ZIP 损坏或缺少必要文件时，打包检查失败，不覆盖上一份有效版本；
- 未配置 AppID 时，README 说明如何使用测试号或自己的 AppID；
- 演示模式初始化失败时，允许重置演示数据；
- 云环境未配置时，阻止切换到云开发模式并显示缺失项；
- 云函数调用失败时，显示明确错误，不自动写入本地演示数据掩盖问题。

## 11. 测试与验收

### 11.1 自动检查

- ZIP 中存在 `project.config.json`、`app.json`、主要页面、组件和云函数；
- ZIP 中不存在 `.git` 和 `project.private.config.json`；
- ZIP 中不存在原 AppID和原云环境 ID；
- 网站下载链接指向真实 ZIP；
- 下载按钮包含正确文件名和下载属性；
- 网站不再公开链接到奶茶浏览器页面；
- 本地演示适配器覆盖登录、商品、购物车、订单和管理操作。

### 11.2 微信开发者工具验收

1. 解压下载包；
2. 使用微信开发者工具导入解压目录；
3. 确认项目可以编译；
4. 检查首页、商品详情、购物车、订单和个人中心；
5. 使用演示账号完成一次下单；
6. 进入管理端更新订单状态和商品状态；
7. 关闭并重新打开项目，确认演示数据仍然存在；
8. 重置演示数据，确认恢复初始状态。

### 11.3 完成标准

只有同时满足以下条件，网站状态才标记为“源码可下载”：

1. 脱敏扫描通过；
2. ZIP 结构检查通过；
3. 网站下载链接真实可用；
4. 微信开发者工具编译通过；
5. 本地演示核心流程通过；
6. README 导入与云开发配置步骤完整。

## 12. 后续扩展

独立域名和公网部署确定后，只需将同一 ZIP 放入最终静态资源目录，不改变小程序源码结构。后续可增加版本号、更新日志和文件校验值，但本阶段不增加自动更新功能。
