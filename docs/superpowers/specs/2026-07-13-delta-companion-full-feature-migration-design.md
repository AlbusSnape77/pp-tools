# Delta Companion 原版功能迁移设计

日期：2026-07-13  
状态：实施中

## 1. 背景

个人网站当前嵌入的 Delta 工具只保留了手动截图识别、浏览器本地花名册和档案编辑。原 `Delta Force` 项目的主要入口并不是本地档案搜索，而是输入昵称或 UID 后自动操作本机游戏客户端，打开玩家资料页、截取四个页面、执行 OCR 并保存档案。

本次改造必须恢复原版完整功能，同时继续使用个人网站现有主题、站内路由和中英日三语。PP Tools 将成为 Delta 功能、运行代码和数据的唯一维护位置；原 `Delta Force` 项目只保留为只读备份。

## 2. 目标

1. 个人网站中的 Delta 与原项目具有相同的核心查询、任务、校准、档案和手动识别能力。
2. 输入昵称或 UID 后，自动操作本机三角洲游戏完成资料采集和 OCR。
3. 新增便携版 `Delta Companion.exe`，为本机网站和未来公网网站提供受控的本机运行能力。
4. 进入 Delta 页面时按运行环境自动启动或唤起 Companion。
5. 原项目现有数据库、花名册、截图数据和校准模板一次性复制到 PP Tools。
6. PP Tools 成为迁移后的唯一数据源，原项目数据不再写入。
7. 网页与 Companion 使用一次性配对码和长期随机令牌建立授权关系。
8. 所有固定界面、状态、错误和校准文案支持中文、英文和日文。

## 3. 非目标

1. 不使用未经授权的远程游戏接口。
2. 不在云端执行游戏自动操作或 OCR。
3. 不把玩家截图、档案或校准图片上传到公网服务器。
4. 不实现两个 Delta 数据目录的双向同步。
5. 不删除原 `Delta Force` 项目及其数据。
6. 本阶段不购买代码签名证书，但保留后续签名能力。
7. 不保证 Chrome 和 Edge 之外浏览器的公网 Companion 连接能力。

## 4. 总体架构

### 4.1 单一源码与数据所有权

原 `Delta Force/dfstats` 中的以下模块迁移到 PP Tools 的 Companion 运行包：

- 游戏客户端自动操作。
- 桌面截图与模板定位。
- OCR、页面分类和档案合并。
- 自动查询任务队列、限速、取消和进度。
- 玩家档案数据库。
- 校准截图、裁剪模板和管理接口。

迁移完成后，PP Tools 的 Companion 源码和 Delta 数据目录是唯一可写来源。原项目保留不变，只用于迁移核对和紧急回退。

### 4.2 组件边界

系统分为四个独立组件：

1. **Delta 网页界面**：负责三语界面、配对、任务展示、档案操作和校准交互。
2. **Delta Companion**：负责授权、本机接口、游戏自动操作、OCR、任务队列、数据库和校准。
3. **个人网站本机服务**：本机模式下检测并启动 Companion，同时继续提供个人网站静态资源。
4. **公网静态网站**：不保存 Delta 私有数据，只连接访问者本机的 Companion。

网页不直接包含桌面控制代码。Companion 不负责个人网站页面渲染。

### 4.3 本机通信

Companion 只监听 `127.0.0.1`，不绑定 `0.0.0.0`，不允许局域网访问。网页通过版本化本机接口通信，例如：

- `GET /api/v1/health`
- `POST /api/v1/pair`
- `POST /api/v1/auto-lookup`
- `GET /api/v1/jobs/{id}`
- `POST /api/v1/jobs/{id}/cancel`
- `GET /api/v1/players`
- `GET /api/v1/players/{id}`
- `PUT /api/v1/players/{id}`
- `DELETE /api/v1/players/{id}`
- `POST /api/v1/manual-lookup`
- `GET /api/v1/calibration`
- `POST /api/v1/calibration/{name}`
- `DELETE /api/v1/calibration/{name}`

接口返回稳定状态代码，网页再根据当前语言渲染文字；Companion 返回的错误不得依赖固定中文字符串做业务判断。

## 5. 启动与协议唤起

### 5.1 本机个人网站

用户双击桌面的“打开我的网站”后，只启动个人网站服务。首次进入 Delta 页面时，个人网站服务检测 Companion；若未运行，则启动 Companion，等待健康检查通过后再连接。

重复进入 Delta 不得启动多个 Companion 进程。Companion 使用单实例锁，并把已有实例置于可连接状态。

### 5.2 公网个人网站

公网 Delta 页面先请求 Companion 健康检查：

1. 已运行：进入连接或配对流程。
2. 未运行但已安装协议：调用 `delta-stats://start`，浏览器确认后启动 Companion。
3. 未安装：显示便携 EXE 下载、运行和重新检测按钮。

首次运行便携 EXE 时，在当前 Windows 用户范围注册 `delta-stats://`。注册不要求管理员权限，不安装系统服务。Companion 提供取消协议注册的操作。

## 6. 配对与安全

### 6.1 首次配对

1. Companion 生成 6 位数字配对码。
2. 配对码在 Companion 窗口显示，有效期 5 分钟且只能成功使用一次。
3. 用户在网页输入配对码。
4. Companion 验证成功后生成高强度随机访问令牌。
5. 浏览器按网站来源保存令牌，Companion 保存令牌摘要和对应来源。
6. 后续连接自动使用令牌，不重复输入配对码。

Companion 支持撤销单个来源和撤销全部配对。撤销后旧令牌立即失效。

### 6.2 请求授权

除健康检查、版本信息和配对提交外，所有接口必须同时验证：

- 有效访问令牌。
- 已配对的精确 `Origin`。
- 允许的方法和请求头。
- 合法的接口版本。

Companion 默认允许本机个人网站来源。未来公网域名确定后加入精确白名单，不使用 `*`。预检请求只对允许来源返回授权头。

### 6.3 浏览器本地网络权限

公网 HTTPS 页面访问 loopback 服务时，按 Chrome 和 Edge 的本地网络访问规则申请权限。界面必须区分：

- Companion 未运行。
- 本地网络权限被拒绝。
- CORS 来源不匹配。
- 配对令牌失效。
- Companion 版本不兼容。

前端优先使用浏览器支持的 loopback 地址空间声明；不支持时使用普通 loopback 请求并显示兼容性说明。

## 7. 数据迁移

### 7.1 迁移来源与目标

来源为原 `Delta Force/data` 和校准目录。目标为 PP Tools 的 Companion 数据目录，至少包含：

- 玩家档案数据库。
- 校准模板。
- Companion 配置。
- 自动备份。
- 临时截图目录。

### 7.2 一次性迁移流程

1. 检测迁移标记；已完成则跳过。
2. 检测原项目路径和数据文件。
3. 在 PP Tools 目标目录创建带时间戳的迁移备份。
4. 复制数据库与校准模板到临时目录。
5. 校验文件大小、SHA-256、数据库结构和必要表。
6. 原子替换目标数据。
7. 写入迁移清单、来源路径、文件校验值和完成标记。
8. 启动后执行只读抽样，确认历史花名册和校准模板可用。

迁移失败时不得修改目标正式数据，不得删除来源数据。用户可以重新执行迁移。

## 8. 自动查询流程

### 8.1 查询启动

顶部输入框恢复为原版自动查询主入口。用户输入昵称或 UID 后点击查询：

1. 验证 Companion、配对、版本、游戏状态和校准状态。
2. 显示 5 秒倒计时和取消按钮。
3. 倒计时结束后提交单工作线程任务。
4. 明确提示鼠标和键盘正在由任务临时接管。

### 8.2 任务步骤

页面实时展示以下稳定步骤：

- 准备游戏窗口。
- 打开社交与添加好友入口。
- 输入昵称或 UID。
- 触发搜索。
- 打开第一个匹配结果。
- 截取玩家首页。
- 截取数据总览。
- 切换并截取排位数据。
- 截取历史战绩。
- 本机 OCR。
- 合并并保存档案。

任务完成后刷新花名册并打开新档案。任务失败或取消后立即停止自动操作，释放鼠标和键盘控制。

### 8.3 任务限制

Companion 保留原版单工作线程、查询间隔、每日查询上限和任务队列。页面显示今日用量、排队位置和限速等待时间。

## 9. 手动识别与档案

手动上传四张截图继续作为备用入口，并改用 Companion 的原版识别和数据库接口。浏览器不再把花名册只保存在 `localStorage`。

档案功能包括：

- 按昵称或 UID 搜索。
- 查看总览、排位、雷达图和最近战绩。
- 修改昵称、标签、备注和 KD。
- 保存和删除档案。
- 复制 UID。
- 显示更新时间。

迁移完成后，历史档案与新增档案都从 Companion 数据库读取。

## 10. 站内校准

校准页面使用站内路由 `#software/delta-force/calibration`，不打开原项目独立页面。功能包括：

- 获取当前桌面截图。
- 选择模板类型。
- 可视化裁剪区域。
- 保存模板。
- 删除模板。
- 显示模板完整性与更新时间。
- 重新执行单项校准。

网页只向 Companion 发送用户明确选择的裁剪图。校准截图不上传公网。

## 11. 页面与状态设计

Delta 页面保留现有个人网站顶部导航和 PP Tools 详情视觉，同时恢复原版操作结构：

- 顶部：自动查询输入、查询、停止、今日用量、校准。
- 状态区：Companion、配对、游戏、校准和当前任务状态。
- 备用区：手动上传截图。
- 左侧：后端花名册筛选与选择。
- 右侧：完整玩家档案。

公网未连接时显示分步骤连接面板：下载 EXE、运行、重新检测、输入配对码、授权本地网络。已连接后不再占用主要界面空间。

## 12. 错误处理

必须区分并提供明确恢复动作：

- Companion 未安装：下载并运行。
- Companion 未运行：协议唤起或手动运行。
- 本地网络权限拒绝：显示浏览器恢复说明。
- 配对码错误或过期：生成新码后重试。
- 令牌失效：重新配对。
- 版本过旧：下载新版 Companion。
- 游戏未启动：启动游戏后重新检测。
- 玩家不存在：修改昵称或 UID。
- 自动定位失败：进入对应校准项。
- OCR 空结果：保留截图并允许手动识别。
- 部分字段成功：保存可用数据并显示警告。
- 达到每日限制：显示下次可用时间。
- 用户取消：立即停止并恢复控制。
- Companion 异常退出：结束网页忙碌状态并保留任务上下文。

## 13. 三语要求

网页所有固定可见文案、可访问名称、状态、配对流程、倒计时、校准、任务步骤和错误说明使用集中式中文、英文、日文字典。

玩家昵称、UID、地图名、游戏段位、截图文字和识别原始数据不做翻译。Companion 返回稳定代码和原始数据，网页负责翻译。

Companion 本机窗口至少支持中文；协议、错误代码和配置字段使用稳定英文标识符。

## 14. Companion 构建与发布

Companion 使用 PP Tools 内的 Python 源码构建单文件 Windows EXE。构建产物包含自动操作、OCR、数据库和校准所需运行库。

发布内容包括：

- `Delta Companion.exe`
- 版本信息
- SHA-256 校验值
- 简短使用说明
- 支持的网页来源配置

初期未签名版本可能触发 Windows SmartScreen。界面不得绕过系统安全提示；后续可使用正式代码签名证书降低提示频率。

## 15. 测试策略

### 15.1 Companion 单元测试

- 配对码生成、过期、单次使用和重放拒绝。
- 令牌生成、摘要存储、撤销和来源绑定。
- CORS、预检、方法限制和未授权请求。
- 单实例、协议注册、协议唤起和退出。
- 任务提交、排队、进度、取消、失败和完成。
- 查询间隔、每日限制和时间边界。
- 数据库增删改查和手动识别。
- 校准模板保存、删除和完整性。
- 数据迁移、校验、重复执行和失败回滚。

### 15.2 网页自动测试

- Companion 未安装、未运行、待配对、已连接和版本过旧状态。
- 5 秒倒计时、取消、任务进度和停止。
- 昵称与 UID 查询。
- 后端花名册和完整档案操作。
- 手动截图识别。
- 站内校准路由和操作。
- 中英日三语键一致性。
- 浏览器前进、后退、刷新和离开页面。

### 15.3 集成与真实验收

- 原项目数据完整迁移到 PP Tools。
- 本机个人网站进入 Delta 后自动启动 Companion。
- 公网 HTTPS 页面通过协议唤起 Companion。
- Chrome 与 Edge 完成权限、配对和后续自动连接。
- 使用真实游戏完成一次昵称查询和一次 UID 查询。
- 查询期间验证倒计时、步骤、停止和鼠标归还。
- 验证四张截图、OCR、档案和原版结果一致。
- 验证校准、手动上传和历史花名册。
- 验证页面和 Companion 退出后的资源释放。

真实游戏自动操作验收需要用户提前登录并打开三角洲游戏，不由自动测试模拟账号登录。

## 16. 验收标准

1. 个人网站 Delta 的核心功能与原项目一致。
2. 顶部查询执行自动游戏查询，不再只是浏览器本地搜索。
3. 昵称与 UID 查询均可完成四页截图、OCR 和档案保存。
4. 手动上传、任务停止、查询限制、校准和档案管理可用。
5. 原项目历史花名册和校准数据复制成功，原数据未删除。
6. PP Tools 成为唯一可写数据源。
7. 本机模式进入 Delta 时自动启动 Companion。
8. 公网模式可以下载、协议唤起并配对便携 EXE。
9. 未配对网页和非白名单来源无法调用自动操作接口。
10. Chrome 与 Edge 的本地网络权限流程可恢复。
11. 所有固定网页文案支持中文、英文和日文。
12. 自动测试、构建、格式检查和真实浏览器验收通过。
13. 实施过程不自动提交或推送仓库。

## 17. 实施顺序

1. 冻结并备份原 Delta 项目。
2. 抽取 Companion 核心模块和版本化接口。
3. 实现数据迁移和唯一数据目录。
4. 实现配对、令牌、来源白名单和单实例。
5. 实现协议注册和本机启动流程。
6. 恢复网页自动查询、任务和后端花名册。
7. 实现站内校准页面。
8. 构建便携 EXE。
9. 接入个人网站本机启动和公网连接流程。
10. 完成自动测试与真实游戏验收。

## 18. 后续事项

1. 公网域名确定后，把精确 HTTPS 来源加入 Companion 白名单。
2. 评估代码签名证书。
3. 根据 Chrome 和 Edge 本地网络权限变化更新连接兼容层。
4. 如需支持更多浏览器，再单独评估云端中转或扩展方案。
